## Technical Architecture

### Hashing Methodology

**Algorithm:** SHA-256 (FIPS 180-4)  
**Implementation:** Node.js `crypto.createHash('sha256')`  
**Encoding:** Hexadecimal (lowercase)

**Chain Checksum Computation:**

1. Collect all artifact SHA-256 hashes
2. Group by block (10.2 â†’ 10.9)
3. Concatenate hashes in block order (no separators)
4. Compute SHA-256 of concatenated string
5. Result is the chain checksum

**Example:**

```javascript
const hash1 = 'abc123...';
const hash2 = 'def456...';
const concatenated = hash1 + hash2; // No separator
const chainChecksum = crypto.createHash('sha256').update(concatenated).digest('hex');
```

### Signature Mechanism

**Format:** GPG detached ASCII-armored signatures (.asc)  
**Signers:** 2 (A.I.K. and EWA)  
**Signature Target:** BLOCK10.9_CLOSURE.md  
**Verification:** `gpg --verify [signature.asc] BLOCK10.9_CLOSURE.md`

### Certificate Generation

**Library:** PDFKit (Node.js PDF generation)  
**Format:** A4 (210mm Ã— 297mm)  
**Layout:**

- Professional border (double-line)
- Title and subtitle sections
- Metadata table (dates, scope, signers)
- Embedded manifest excerpt (SHA-256 hashes)
- Chain checksum display
- QR code (ledger entry link)

### Verification Process

**Multi-Layer Verification:**

1. **File Integrity:** Re-hash all artifacts, compare with manifest
2. **Chain Integrity:** Recompute chain checksum, verify match
3. **Ledger Integrity:** Validate JSON structure, check references
4. **Signature Integrity:** GPG verify both signatures
5. **Certificate Integrity:** Hash certificate, compare with ledger entry
6. **Cross-Reference:** Verify all artifact paths exist and are correct
